# Security policy

The pipeline-runner project takes security seriously. We appreciate responsible disclosure of vulnerabilities.

## Reporting a vulnerability

Please report security issues by email to **security@pipeline-runner.dev**. Include a clear description of the issue, affected versions if known, and steps to reproduce or assess impact. Do not open a public issue for undisclosed vulnerabilities.

## Response timeline

We aim to acknowledge receipt within **48 hours** and to provide an initial assessment within **7 days**. Critical fixes are prioritized; we will coordinate disclosure timelines with you when a patch is ready.

## Supported versions

Security updates are provided for the latest minor release line of the current major version. Upgrade to the latest release when possible.
